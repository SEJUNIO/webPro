[트랜잭션]
데이터베이스에서 트랜잭션은 데이터 처리의 한단위.
오라클에서 발생하는 여러개의 sql 명령문을 하나의 논리적인 작업 단위로 처리하는 작업.
하나의 트랜잭션은 ALL-OR-NOTHING 방식으로 처리
여러개의 명령어 집합이 정상적으로 처리되면 정상 종료하도록 하며 여러개의 명령어 중에서 하나의 명령어도 잘못되면 전체 취소
데이터베이스에 작업의 단위로 개념이 생긴 이유는 데이터의 일관성을 유지하면서 안정적으로 데이터를 복구시키키 위함 

[트랜잭션] COMMIT, SAVEPOINT, ROLLBACK

[COMMIT]
모든 작업들을 정상적으로 처리하겠다고 확정하는 명령어로 트랜잭션의 처리 과정을 데이터베이스에 모두 반영하기 위해서 변경된 내용 모두 영구 저장합니다.
COMMIT 명령어를 수행하게 되면 하나의 트랜잭션 과정을 종료
(INSERT, UPDATE, DELETE) 작업 내용을 실제로 DB에 저장
이전 데이터가 완전히 업데이트 됩니다.
모든 사용자가 변경된 데이터의 결과를 볼 수 있습니다.

[ROLLBACK]
작업중 문제가 발생되어 트랜잭션의 처리가 과정에서 발생한 변경사항을 취소하는 명령어
롤백 역시 트랜잭션 과정을 종료하게 된다.
트랜잭션으로 인한 하나의 묶음 처리가 시작되기 이전 상태로 되돌립니다.
이전 커밋한곳 까지만 복구.

[커밋 명령어와 롤백 명렁의 장점]
데이터 무결성이 보장, 영구적인 변경 전에 데이터의 변경 사항을 확인할 수 있습니다.
논리적으로 연관된 작업을 그룹화

[SAVEPOINT]
SAVEPOINT 명령을 써서 현재의 트랜잭션을 작게 분할
저장된 SAVEPOINT는 ROLLBACK TO SAVEPOINT문을 사용하여 표시한곳까지 롤백 할수 있습니다.
여러개의 SQL문의 실행을 수반하는 트랜잭션의 경우 사용자가 트랜잭션 중간 단계에서 세이브 포인트 지정.

[SAVEPOINT로 특정위치를 지정 사용형식]
SAVEPOINT LABEL_NAME;

[SAVEPOINT 지정해 놓은 특정위치로 되돌아가기 위한 사용 형식]
ROLLBACK TO LABEL_NAME;
