[시멘틱태그]
일반적으로 html5 태그의 가장 큰 변화를 시멘틱 태그라고 합니다.
기계적인 검색 엔진은 어떠한 태그가 어떠한 기능을 하는지 분별할 수 없고 웹 페이지에서 
데이터를 효율적으로 추출할 수 없다, 이를 해결하고자 특정 태그에 의미를 부여해서 웹페이지를 만드는 시도가 시작 되었다.
그로 이를 시멘틱 웹이라고 표현하며 시멘틱 웹의 장점은 검색엔진이 보다 효율적으로 검색을 할 수 있고 유지보수시 효율적
시멘틱 태그의 구조는
header : 헤더
nav : 내비게이션
aside : 사이드에 위치하는 공간
section : 여러 중심 내용을 감싸는 공간
acticle : 글자가 많이 들어가는 부분
footer : 꼬리말 
이렇게 구성이 되어진다.
